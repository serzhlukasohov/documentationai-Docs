## Overview

Welcome to Serhii Lukashov's documentation space. You use this platform to organize, manage, and share all your project documentation in one centralized location. It supports rich MDX formatting, interactive components, and seamless version control, making it ideal for developers, teams, and solo creators who need efficient doc workflows.

This space helps you create self-contained pages with over 250 words of detailed content, ensuring every document follows best practices like the EPPO principle (Every Page Produces Output). Start exploring key features, follow the quick start guide, and navigate effortlessly to build your knowledge base.

## Quick Start

Get up and running in minutes. Follow these steps to create your first documentation page.

<Steps>
  <Step title="Create a New Page" icon="plus">
    Navigate to the docs folder and add a new `.mdx` file, like `quickstart.mdx`.

    Start with YAML frontmatter:

````mdx
---
title: Quickstart
description: Get started quickly.
---
````
  </Step>
  <Step title="Add Content" icon="edit-3">
    Write H2 headings and embed components. Use backticks for special characters like `{variable}` or `<100ms`.
  </Step>
  <Step title="Preview & Deploy" icon="eye">
    Preview locally with `npm run dev`, then push to deploy. Your site updates instantly.
  </Step>
</Steps>
