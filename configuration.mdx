## Brand Colors and Themes

Set your primary brand color to `#3B82F6` for buttons, links, and accents. Access the theme editor in your dashboard at `https://dashboard.example.com/settings/theme`.

<Steps>
  <Step title="Open Theme Settings" icon="settings">
    Navigate to Dashboard > Settings > Theme.
  </Step>
  <Step title="Enter Brand Color">
    Input your hex code in the Primary Color field.
  </Step>
  <Step title="Preview and Save">
    Preview changes live, then click Save.
  </Step>
</Steps>

For advanced theming, edit `config.json`:

<CodeGroup tabs="JSON,YAML">
  ```json
  {
    "theme": {
      "primaryColor": "#3B82F6",
      "secondaryColor": "#1D4ED8"
    }
  }
  ```
  ```yaml
  theme:
    primaryColor: "#3B82F6"
    secondaryColor: "#1D4ED8"
  ```
</CodeGroup>

<Callout kind="tip">
  Test colors in dark and light modes before saving to ensure readability.
</Callout>

## User Permissions

Define roles to control access. Admins manage everything, editors update content, and viewers read only.

| Role     | Can Edit | Can Delete | Can Invite Users |
|----------|----------|------------|------------------|
| Admin    | Yes     | Yes       | Yes             |
| Editor   | Yes     | No        | No              |
| Viewer   | No      | No        | No              |

<Steps>
  <Step title="Add User" icon="users">
    Go to Settings > Users > Invite.
  </Step>
  <Step title="Assign Role">
    Select role from dropdown and send invite.
  </Step>
</Steps>

Use `<ParamField>`-style config for fine-grained control in `permissions.json`:

```json
{
  "users": [
    {
      "email": "user@example.com",
      "role": "editor",
      "pages": ["docs/*"]
    }
  ]
}
```

## Backup and Version Control

Protect your docs with automatic backups and Git integration.

<Expandable title="Git Setup" default-open="true">
  Connect to `https://github.com/your-repo/docs.git`. Push changes automatically or on schedule.

  ```bash
  git remote add origin https://github.com/your-repo/docs.git
  git push -u origin main
  ```
</Expandable>

<Expandable title="Export Options">
  Download as PDF, HTML, or Markdown ZIP from Dashboard > Backups.
</Expandable>

<Callout kind="alert">
  Enable daily backups to avoid data loss. Review logs at `https://dashboard.example.com/backups`.
</Callout>
